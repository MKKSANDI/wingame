Version 1.1.0 (Updated) - Major Performance & UI Improvements
─────────────────────────────────────────────────────────────────────────────

🎯 MAJOR IMPROVEMENTS:

1. QUEUE-BASED SYSTEM
   ✅ Toggle switches NO LONGER apply immediately
   ✅ Changes are queued and only applied when "Apply All Enabled" is clicked
   ✅ NO MORE LAG when selecting options
   ✅ Much faster and more responsive UI

2. PROGRESS BAR POPUP
   ✅ Modern progress dialog appears during application
   ✅ Shows accurate progress (percentage and current optimization)
   ✅ Non-blocking - smooth animation throughout
   ✅ Clear status messages for each optimization
   ✅ Success/failure count at the end

3. SETTINGS PERSISTENCE
   ✅ All toggle states are saved automatically
   ✅ On launch, previously enabled options are restored
   ✅ Toggle states persist across sessions
   ✅ Config saved to optimizer_config.json

4. PROFESSIONAL UI REDESIGN
   ✅ Removed ALL emojis
   ✅ Added PNG icon support (system.png, game.png)
   ✅ Enhanced Windows 11 Aero theme
   ✅ More subtle colors with rgba transparency
   ✅ Cleaner, more professional appearance
   ✅ Better spacing and layout
   ✅ Smaller, more refined components

NEW CLASSES:
  • OptimizationWorker (QThread)
    → Handles optimization application in background
    → Emits progress signals
    → Non-blocking operation

  • ModernProgressDialog (QDialog)
    → Custom progress dialog with Windows 11 styling
    → Shows percentage and status
    → Cannot be closed during operation

MODIFIED BEHAVIOR:
  • on_toggle() - Now only updates state, doesn't apply
  • apply_all() - Creates worker thread and progress dialog
  • All apply_tweak() calls moved to worker thread

NEW FEATURES:
  • Icon support via QPixmap
  • Progress signals (percentage, status message)
  • Threaded execution
  • Non-blocking UI updates

  🎯 USER WORKFLOW CHANGES
═══════════════════════════════════════════════════════════════════════════════

OLD WORKFLOW (v1.0):
  1. Toggle switch ON
  2. ⏳ Wait while system applies changes (lag!)
  3. ⏳ UI freezes briefly
  4. Toggle another switch
  5. ⏳ Wait again...

NEW WORKFLOW (v1.1):
  1. Toggle switches ON/OFF (instant!)
  2. Toggle more switches (instant!)
  3. Click "Apply All Enabled"
  4. Watch progress bar
  5. ✅ Done! All applied at once

═══════════════════════════════════════════════════════════════════════════════
  💾 SETTINGS PERSISTENCE
═══════════════════════════════════════════════════════════════════════════════

HOW IT WORKS:
  • Every toggle change saves to optimizer_config.json
  • On app launch, config is loaded
  • All toggle switches restore to previous state
  • You see exactly what you had enabled last time

EXAMPLE:
  Session 1:
    - Enable "Power Throttling" ✓
    - Enable "CPU Gaming" ✓
    - Close app
    
  Session 2 (Next launch):
    - App opens
    - "Power Throttling" is already ON ✓
    - "CPU Gaming" is already ON ✓
    - Ready to use!
